KayKit Nature Pack - Export Notes
==================================

These models use a color palette texture (61x33 pixels) where:
- Stone/gray colors: center of texture
- Wood/tan colors: edges at pixels x=1-2 and x=57

Original FBX files have UVs in a non-standard coordinate space (~-19 to +19).
They must be transformed to work with the palette texture.

EXPORT WORKFLOW
---------------

1. Convert ASCII FBX to GLTF using assimp:

   assimp export original/<model>.fbx <model>.gltf

2. Fix UVs in Blender (UVs are centered at origin, need to map to 0-1):

   SCALE_U = 38.5
   SCALE_V = 25.0

   newU = oldU / SCALE_U + 0.5
   newV = oldV / SCALE_V + 0.5

   This maps:
   - Original U range (-18.7 to +18.7) -> (0.015 to 0.985)
   - Original V range (-13.4 to +10.0) -> (-0.04 to 0.90)

3. Copy texture from original folder:

   cp original/<model>.png <model>.png

4. Generate metadata:

   python3 mcp_tools/import.py <model>.gltf <model>.png

5. Create material linking texture (use textured shader)

BLENDER SCRIPT
--------------

import bpy

input_path = "path/to/model.gltf"
bpy.ops.wm.read_factory_settings(use_empty=True)
bpy.ops.import_scene.gltf(filepath=input_path)

SCALE_U = 38.5
SCALE_V = 25.0

for obj in bpy.context.scene.objects:
    if obj.type != 'MESH':
        continue
    mesh = obj.data
    if not mesh.uv_layers:
        continue
    uv_layer = mesh.uv_layers.active
    for loop in mesh.loops:
        uv = uv_layer.data[loop.index].uv
        uv[0] = uv[0] / SCALE_U + 0.5
        uv[1] = uv[1] / SCALE_V + 0.5

bpy.ops.export_scene.gltf(
    filepath=input_path,
    export_format='GLTF_SEPARATE',
    export_texcoords=True,
    export_normals=True,
    export_materials='NONE'
)
